'save' in 'npm install' 2020-05-03

当我们使用npm包管理时，我们的项目目录中必然会存在一个名为 ‘node_modules’ 的文件夹，它的作用就是帮助我们保存开发时所需要用到的依赖包，可是当我们需要下载依赖包时，我们会遇到如下指令

会将 MOUDLE_NAME 包安装到 node_modules 目录中；
不会修改package.json；
之后运行 npm install 命令，不会自动安装 MOUDLE_NAME 包，其原因就是没有在 package.json 中注册；

会把 MOUDLE_NAME 包安装到node_modules目录中；
会在package.json的dependencies属性下添加 MOUDLE_NAME；
之后运行npm install命令时，会自动安装 MOUDLE_NAME 到node_modules目录中；
之后运行npm install –production或者注明NODE_ENV变量值为production时，会自动安装msbuild到node_modules目录中；
如果在项目中安装的 MOUDLE_NAME 包 需要在生产环境中依赖 则使用 –save，比如 jQuery。

会把 MOUDLE_NAME 包安装到node_modules目录中；
会在package.json的devDependencies属性下添加 MOUDLE_NAME；
之后运行npm install命令时，会自动安装 MOUDLE_NAME 到node_modules目录中；
之后运行npm install –production或者注明NODE_ENV变量值为production时，不会自动安装 MOUDLE_NAME 到node_modules目录中；
如果在项目中安装的 MOUDLE_NAME 包 不需要在生产环境中依赖 则使用 –save-dev，比如babel。
